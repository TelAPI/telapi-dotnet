InboundXML

We've made generating InboundXML with this library as intuitive as possible by defining a simple code design pattern which corresponds to the structure of the XML. The best way to understand this pattern is to take a look at an example:

using System;
using TelAPI.InboundXML;
using TelAPI.InboundXML.Enum;

namespace TelAPI.InboundXML.Example
{
    public class SayExample
    {
        static void Main(string[] args)
        {
            var response = new Response();
            Console.WriteLine("{0}", response.CreateXml());
        }
    }
}

/*
</Response>
*/

First and foremost, you need to instantiate a new Response object. This assignment alone begins the process of generating the xml. As shown above, if we output response.CreateXml(), the start of the base elements required for an InboundXML document are printed to the console.

using System;
using TelAPI.InboundXML;
using TelAPI.InboundXML.Enum;

namespace TelAPI.InboundXML.Example
{
    public class SayExample
    {
        static void Main(string[] args)
        {
            var response = new Response();

            response.Record("http://foo-url.com/");

            Console.WriteLine("{0}", response.CreateXml());
            Console.ReadKey();
        }
    }
}

/*
<Response>
  <Record action="http://foo-url.com/" />
</Response>
*/

Creating elements is super easy. Simply reference the element you would like to add to the XML document and append them to the response with dot notation as shown above.

using System;
using TelAPI.InboundXML;
using TelAPI.InboundXML.Enum;

namespace TelAPI.InboundXML.Example
{
    public class SayExample
    {
        static void Main(string[] args)
        {
            var response = new Response();

            response.Record("http://example.com/")
                    .Say("Hello!", Voice.man, null);

            Console.WriteLine("{0}", response.CreateXml());
            Console.ReadKey();
        }
    }
}

/*
<Response>
  <Record action="http://foo-url.com/" />
  <Say voice="man">Hello!<Say />
</Response>
*/

This dot chaining can continue for as many elements as needed in the response. Specifying of element attributes is achieved by passing them to to the element method. For details on element methods and which parameters are required when, see the InboundXML source here.

using System;
using TelAPI.InboundXML;
using TelAPI.InboundXML.Enum;

namespace TelAPI.InboundXML.Example
{
    public class SayExample
    {
        static void Main(string[] args)
        {
            var response = new Response();

            response.Record("http://example.com/")
                    .Say("Hello!", Voice.man, null);
                    .Dial(new Dial().Conference("ConfRoom"));

            Console.WriteLine("{0}", response.CreateXml());
            Console.ReadKey();
        }
    }
}

/*
<Response>
  <Record action="http://foo-url.com/" />
  <Say voice="man">Hello!<Say />
  <Dial>
    <Conference>ConfRoom<Conference/>
  <Dial />
</Response>
*/

Children elements, such as Conference, can be added to the document by nesting them within the desired parent elements, as shown above.

REST API

Before using any of the TelAPI REST methods available, you must authenticate your account. Do so by including your REST API credentials in the following way:

var client = new TelAPIRestClient(
    "{AccountSid}", 
    "{AuthToken}"
);

Notice the code above instantiates and assigns a client object. Every method used throughout this helper library will use this main client class.

Below are usage examples and additional information for each function of TelAPI accomplished via REST requests. If you need to dig deeper, the source defining each REST method is found here.

Make Call

Documentation

Make a post request to place a call with the MakeCall method:

try
{
    var callOptions = new CreateCallOptions();

    var call = client.MakeCall(
        "{from-number}", 
        "{to-number}", 
        "{action-url}"
    );
    Console.WriteLine("Call sid : {0}", call.Sid);
}
catch (TelAPIException ex)
{
    Console.WriteLine("{0}", ex.Message); 
}

If any optional parameters are being used when making the call, they must be specified with the CallOptions method. The options are then passed to the MakeCall method to place the call:

try
{

    var callOptions = new CallOptions();
    
    callOptions.From = "7325551234";                         //string, required
    callOptions.To = "7325551234";                           //string, required
    callOptions.Url = "http://some-url.com";                 //string, required
    callOptions.Method = HttpMethod.Post;                    //HttpMethod, optional
    callOptions.Timeout = 5;                                 //int, optional
    callOptions.SendDigits = "www123";                       //string, optional
    callOptions.StatusCallback = "http://some-url.com";      //string, optional
    callOptions.StatusCallbackMethod = HttpMethod.Get;       //HttpMethod, optional
    callOptions.FallbackUrl = "http://some-url.com";         //string, optional
    callOptions.FallbackMethod = HttpMethod.Get;             //HttpMethod, optional
    callOptions.HeartbeatUrl = "http://some-url.com";        //string, optional
    callOptions.HeartbeatMethod = HttpMethod.Post;           //HttpMethod, optional
    callOptions.ForwardedFrom ="7325551234";                 //string, optional
    callOptions.HideCallerId = false;                        //boolean, optional
    callOptions.Record = true;                               //boolean, optional
    callOptions.RecordCallback = "http://some-url.com";      //string, optional
    callOptions.RecordCallbackMethod = HttpMethod.Post;      //HttpMethod, optional
    callOptions.Transcribe = true;                           //boolean, optional
    callOptions.TranscribeCallback = "http://some-url.com";  //string, optional

    var call = client.MakeCall(callOptions);

    Console.WriteLine("Call sid : {0}", call.Sid);
}
catch (TelAPIException ex)
{
    Console.WriteLine("{0}", ex.Message); 
}
